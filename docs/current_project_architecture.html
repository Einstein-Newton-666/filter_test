filter_test 当前架构与计算原理

这页只描述当前代码正在做什么。重点是两件事：仿真器如何从真实几何生成 detector 风格观测， typed GTSAM 图优化如何把像素观测、运动模型和车辆/风车几何放进同一张图。

当前架构 图优化架构 因子图原理 GTSAM 指南 ROS2 版本参考
边界先说清楚： 装甲板图优化是当前主链路；前哨站已有三装甲板仿真、传统滤波观测模型和 typed graph 三槽位分支； 能量机关图优化已经有 center/roll/vroll/normal_yaw 图模型，并把图优化后的连续 roll 序列送入大符 Ceres 曲线拟合器。

1. 总链路

装甲板 armor_simulation_node 生成带噪 PnP 观测，graph_optimizer_test 做纯 typed GTSAM 优化；filter_graph_optimizer 可用滤波器结果作为前端初值。
能量机关 rune_simulation_node 发布不带物理编号的紧凑五点观测，rune_graph_optimizer 内部匹配 slot 并优化 roll 状态。
相机外参 gimbal_simulation 发布 TF；仿真器和图优化器运行时查 odom -> camera_optical_frame。
通用图 core auto_graph::GraphOptimizer 只管理 typed 变量、key、因子和 GTSAM 生命周期，不放 ROS 业务逻辑。

装甲板链路

仿真真值目标运动、standard/outpost 几何
视觉观测投影、像素噪声、IPPE PnP
滤波/图优化纯图优化，或 filter 前端 + graph 后端
输出/tracker/target 和 marker
启动提醒： filter_test.launch.py 会读取 config.yaml 顶层 tracker_backend，支持 graph 和 filter_graph。 当前 launch 返回列表实际启用仿真、云台和 filter； 图优化后端、angle_solver、能量机关和 jlu 节点仍可能以注释形式保留。 使用前需要确认对应节点已加入 LaunchDescription，或者直接用 ros2 run 单独启动。

能量机关链路

roll 运动small 固定角速度，big 随机曲线
紧凑观测只发布 active blade 五点，不带物理编号
RuneCvGraph优化 center/roll/vroll/normal_yaw
输出RuneInfo、TargetPose、marker

2. 仿真器计算原理

仿真器不直接给 3D pose 加噪声。它先用真实 3D 几何投影出像素点，再给像素点加噪声， 最后用 PnP 反解出 detector 风格的观测。这样误差传播更接近真实视觉链路。

运动积分
x   = x   + vx   * dt + 0.5 * ax   * dt^2
y   = y   + vy   * dt + 0.5 * ay   * dt^2
yaw = yaw + vyaw * dt + 0.5 * ayaw * dt^2

vx   = vx   + ax   * dt
vy   = vy   + ay   * dt
vyaw = vyaw + ayaw * dt

过程噪声按 a_eff = a + sqrt(q / dt) * N(0,1) 注入。

标准装甲板位置
armor_yaw_i = yaw + i * pi/2
r_i         = (i % 2 == 0) ? r1 : r2
z_i         = (i % 2 == 0) ? z1 : z2

position_i = [
  x - cos(armor_yaw_i) * r_i,
  y - sin(armor_yaw_i) * r_i,
  z_i
]

局部角点顺序固定为 [左下, 左上, 右上, 右下]，局部坐标为 X=法线, Y=宽度, Z=高度。

前哨站三装甲板
armor_yaw_i = yaw + i * 2*pi/3
local_i = [
  -radius * cos(armor_yaw_i),
  -radius * sin(armor_yaw_i),
  dz_i
]

position_i = center + body_rotation * local_i
R_i = body_rotation * Rz(armor_yaw_i) * Ry(outpost.armor_pitch)

dz_i = [0, dz_1, dz_2]。图优化侧使用同样的 3 槽位/120 度模型，仿真器通过 Armor.priority 传递物理 index。

相机投影
p_cam = T_camera_odom * p_odom
x_n = X_cam / Z_cam
y_n = Y_cam / Z_cam

r2 = x_n^2 + y_n^2
radial = 1 + k1*r2 + k2*r2^2

u = fx * x_distorted + cx
v = fy * y_distorted + cy

畸变模型是 Brown-Conrady：径向 k1/k2 加切向 p1/p2。

像素噪声
sigma(d) = sigma_min + k * (d - d_opt)^2

p_i' = p_i + n_common + n_i
n_common ~ N(0, (rho*sigma)^2)
n_i      ~ N(0, (sqrt(1-rho^2)*sigma)^2)

rho 越大，角点越像一起平移，局部矩形形状破坏越小；单点总方差仍保持 sigma^2。

掉检概率
area_factor = sigmoid(k_area * (area - min_area))
dist_factor = sigmoid(k_dist * (max_distance - distance))
detect_prob = detection_probability * area_factor * dist_factor

距离太远、面积太小或随机 miss 都会导致该目标不发布。

PnP 观测
noisy_pixels + armor_3d_model
  -> cv::solvePnP(..., SOLVEPNP_IPPE)
  -> R_cam_armor, t_cam_armor
  -> 转回 odom
  -> /detector/armors

平面 IPPE 仍可能有双解，仿真侧用 yaw 先验做分支修正。

3. 能量机关仿真原理

能量机关以叶片局部五点为模型，不移植 detector。仿真节点每帧计算真实 5 片叶片， 但检测话题只发布当前 active 且通过可见性/掉检判断的叶片；物理 slot 只保留在 ground truth 和 marker 中，图优化内部通过 PnP roll/center 匹配推断。 

r_center = (0,  0.000, 0.0000)
near     = (0,  0.000, 0.5415)
left     = (0,  0.160, 0.8585)
far      = (0,  0.000, 0.7000)
right    = (0, -0.160, 0.8585)
叶片世界坐标
R_base = Rz(normal_yaw) * Ry(normal_pitch)
R      = R_base * Rx(roll)

point_world = center + R * point_local
小符
roll = roll + angular_velocity * dt
观测与可视化
small: active_blades = 1 random blade
big:   active_blades = 2 random blades
switch interval = 3s

/rune_detector/rune_targets: compact observed active blades
/rune_simulation/ground_truth: center x/y/z + normal_yaw/normal_pitch + roll/vroll + mode + active_blades
/rune_simulation/marker:
  rune_center     center sphere
  rune_target     target sphere
  rune            5 fitted blade spheres + radius lines
  observed_position observed blade spheres
  rune_status     roll/obs text

RuneInfo: normal_yaw/normal_pitch + roll/vroll/curve params
TargetPose.orientation: RPY(0, normal_pitch, normal_yaw)
大符
v(t) = dir * abs(bias + amplitude * sin(frequency*t + phase) + noise)
roll = roll + v(t) * dt

图优化节点先用常速 roll 图平滑单帧观测，再把连续 roll 序列喂给 5 参数大符曲线。

4. 外参和 TF

当前项目没有使用 auto_aim 的 xacro/URDF 外参配置文件。相机外参由 gimbal_simulation 直接广播。 

odom
  -> gimbal_link              # 动态 yaw/pitch
  -> camera_link              # camera_gimbal_tx/ty/tz
  -> camera_optical_frame     # 固定 RPY = -pi/2, 0, -pi/2

仿真器查到的 TF 是：

lookupTransform("odom", "camera_optical_frame")

p_odom = R_c2o * p_cam + t_c2o

投影需要反方向，所以代码中转换为：

R_o2c = R_c2o^T
t_o2c = -R_c2o^T * t_c2o
p_cam = R_o2c * p_odom + t_o2c

5. 装甲板图优化原理

图优化不是直接拟合单帧 PnP pose。每块匹配装甲板引入一个辅助 Pose3， 像素误差约束辅助 pose，车辆几何误差再把辅助 pose 拉回整车状态。

状态变量

变量类型动态含义
centerPoint3是目标中心位置
velocityVector3是中心速度
yawRot2是目标朝向
vyawdouble是yaw 角速度
radius_a/radius_bdouble否奇偶装甲板半径，无界 logistic 状态
dzdouble否奇偶装甲板高度差

因子拆分

因子残差含义为什么需要
TranslationFactor center_k - center_{k-1} - velocity_{k-1} * dt 常速平移模型。
YawFactor yaw_k = yaw_{k-1} + vyaw_{k-1} * dt 用 Rot2 处理角度跨 +-pi。
ArmorTypedReprojFactor 辅助 Pose3 投影角点到像素，与观测像素相减。 把真实 detector 误差放在像素层，避免只相信 PnP pose。
ArmorRadiusCenterZFactor / ArmorRadiusDZFactor 辅助 pose 的 odom 位置/yaw 与整车中心、半径、高度差一致。 把单块装甲板观测融合成整车状态。
ArmorEdgeCenterZReprojFactor / ArmorEdgeDZReprojFactor 从 center/yaw/radius/dz 直接预测四角点，对四条边计算 auto_aim 风格重投影残差。 让主状态直接看到 2D 边缘几何，补强 yaw、半径和中心位置约束。

direct edge 残差不替代角点像素因子。它参考 auto_aim EdgeLoss： 每条边先算投影边和观测边的顶点位移、边长差和方向角差，再按装甲板相对相机的倾斜角混合。 

pixel_error = (0.5*(|p_i-o_i| + |p_j-o_j|) + |len_proj-len_obs|) / len_obs
angular_error = atan2(cross(edge_proj, edge_obs), dot(edge_proj, edge_obs))
cost = pixel_error^2 * |sin(inclined)| + angular_error^2 * |cos(inclined)| * edge_loss_slope_k
residual_edge = cost^(1/4)

偶数装甲板 direct factor 连接 radius_a/yaw/center； 奇数装甲板多连接 dz。调参入口是 use_edge_reproj_factor、edge_reproj_sigma 和 edge_loss_slope_k。

匹配代价当前是：

cost = yaw_diff + 3 * position_error

单帧内匹配保证装甲板 index 唯一；未匹配观测标记为 -1 并跳过写图。

6. 能量机关图优化原理

RuneCvGraph 复用同一个 auto_graph::GraphOptimizer core。 它的图比装甲板小：中心、roll、roll 速度和一个静态平面法线 yaw。

变量类型动态含义
centerPoint3是能量机关中心
rollRot2是当前叶片角度
vrolldouble是roll 角速度
normal_yawRot2否风车平面法线 yaw
因子残差含义
RuneRollFactor roll_k = roll_{k-1} + vroll_{k-1} * dt
RuneVrollFactor vroll_k - vroll_{k-1}，随机游走。
RuneBladeReprojFactor 辅助叶片 Pose3 将五个局部点投影到像素，与匹配到该内部 slot 的五点观测相减。
RuneBladeGeometryFactor 辅助叶片 pose 与 center/roll/normal_yaw 的位置、roll 和 yaw 一致。
RuneBladeDirectReprojFactor 直接连接 center/roll/normal_yaw，用四个扇叶点的两条对角线角度差和归一化长度差做残差。

发布预测时优先使用大符曲线；拟合数据不足时退回常速外推：

angle(t) = -a*cos(w*(t+t0)) + b*t + c
predicted_roll = curve_ready ? angle(t + predict_dt)
                             : optimized_roll + vroll * predict_dt
