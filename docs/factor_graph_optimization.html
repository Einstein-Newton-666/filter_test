1. 本质：非线性最小二乘

所有优化问题都可以写成：

x* = argmin  Σ  || errorᵢ(x) ||²_Σᵢ
       x       i
  • errorᵢ(x) — 第 i 个约束的残差（预测和观测的差）
  • Σᵢ — 该约束的信息矩阵（协方差的逆），权重大的约束信息多
  • 目标：找到 x 使得所有残差的加权平方和最小
类比：想象用橡皮筋连接变量和观测。每根橡皮筋的拉力 ∝ 残差 × 信息矩阵。优化 = 找到所有橡皮筋拉力平衡的位置。

3. Gauss-Newton 迭代

非线性最小二乘不能直接解——需要在当前估计值 x₀ 处 Taylor 展开：

error(x₀ + Δx) ≈ error(x₀) + J(x₀) · Δx
                  \______/   \_________/
                  残差 b      雅可比 J

目标: min || J·Δx + b ||²
       Δx

正规方程 (Normal Equations):
    JᵀJ · Δx = -Jᵀb
    \__/     \____/
    近似       梯度
    Hessian H

每轮迭代：计算 J 和 b → 组装 H 和 g → 解 H·Δx = -g → x₁ = x₀ + Δx → 重复。

具体例子：YawMotionFactor

状态: r_pre=Rot2(0.5), w_pre=2.0, dt=0.01, r_cur=Rot2(0.53)

预测: r_pred = r_pre * Rot2(w_pre·dt) = Rot2(0.52)
残差: error = r_pred.localCoordinates(r_cur) = 0.53 - 0.52 = 0.01

雅可比 (1×1 对每个变量, 用 Rot2 流形):
  ∂error/∂r_pre = -1
  ∂error/∂w_pre = -0.01
  ∂error/∂r_cur = +1

正规方程 (3×3):
  JᵀJ · [Δr_pre, Δw_pre, Δr_cur]ᵀ = -Jᵀ · 0.01
关键：Rot2 流形 — 用 localCoordinates 而不是欧几里得减法。yaw=3.14 和 yaw=-3.14 是同一点，localCoordinates 返回 0 而非 6.28。

手写 GTSAM 因子的内部结构 — 以 ArmorRadiusDZFactor 为例

每个手写 GTSAM 因子的 evaluateError 分为三层：

evaluateError(armor_pose_camera, radius, dz, center_yaw, center_pos)
│
├─ 第一层: 坐标系变换 + 几何量提取
│   pose = armor_pose_camera → T_camera_to_odom → armor_pose_odom
│   armor_yaw     = rotationMatrixToRPY(R_odom).z()    ← 旋转矩阵→RPY→取yaw
│   armor_position = armor_pose_odom.translation()
│   radius_b       = logisticFunction(radius)           ← logistic→物理半径
│   nx,ny = (cos,sin)(armor_yaw)    ← 装甲板法线方向
│   tx,ty = (-ny, nx)               ← 装甲板切线方向 (法线逆时针90°)
│   dx,dy = center - armor           ← 中心→装甲板向量
│
├─ 第二层: 4D 误差计算
│   tangential_err = tx·dx + ty·dy           ← 切线投影,应≈0
│   radial_err     = nx·dx + ny·dy - r_b     ← 法线投影,应=半径
│   z_err          = center_z + dz - armor_z
│   pred_yaw       = center_yaw + index·(2π/4)
│   yaw_err        = armor_yaw.localCoordinates(pred_yaw)  ← SO(2)包裹
│
└─ 第三层: 雅可比矩阵 (链式法则)
    H1 (armor_pose, 4×6) = [J_rot(4×3) | J_trans(4×3)]
      ├─ J_p (4×3) = ∂error/∂armor_pos            ← 位置雅可比
      ├─ d_e_d_psi (4×1) = ∂error/∂armor_yaw_angle ← 角度雅可比
      ├─ d_psi_d_omega (1×3) = ∂yaw/∂rotation_vector
      │     = [0, sin(roll)/cos(pitch), cos(roll)/cos(pitch)]
      │     ↑ 需要 rotationMatrixToRPY 提供 roll 和 pitch 作为中间变量
      ├─ J_rot = d_e_d_psi × d_psi_d_omega          ← 旋转部分 (4×3)
      └─ J_trans = J_p × R_odom                     ← 平移部分 (4×3)
    H2 (radius)  = [0, -dlog, 0, 0]ᵀ             ← logistic导数
    H3 (dz)      = [0, 0, 1, 0]ᵀ
    H4 (yaw)     = [0, 0, 0, 1]ᵀ
    H5 (pos)     = [tx,ty,0; nx,ny,0; 0,0,1; 0,0,0]
rotationMatrixToRPY 在雅可比链中的角色： GTSAM 的 Pose3 使用右乘扰动 R' = exp(ω) × R。 yaw 角是 ZYX 欧拉角对旋转矩阵的偏导数，需要 roll 和 pitch 作为中间变量： ∂yaw/∂ω = [0, sin(roll)/cos(pitch), cos(roll)/cos(pitch)]。

这个透传是必需的：Pose3 通过右乘扰动暴露 ω 方向上的梯度， 而因子关心的是 yaw 角的变化。rotationMatrixToRPY 提供 roll/pitch， d_psi_d_omega 用它们完成 ω→yaw 的链式求导。

如果跳过这一步、硬编码 yaw 或忽略旋转雅可比， 优化器会在错误的梯度方向更新 armor pose → 收敛失败。

4. 多因子拼接：组装大矩阵

单帧可能有 10+ 个因子。每个因子贡献 ||J_i·Δx + b_i||²。全部堆叠：

    ┌        ┐        ┌    ┐
    │  J₁    │        │ b₁ │       J₁ 来自 YawMotionFactor (1×3)
J = │  J₂    │    b = │ b₂ │       J₂ 来自 VyawSmoothFactor (1×2)
    │  ...   │        │ ...│       J₃ 来自 ArmorReprojFactor (2×6)
    └        ┘        └    ┘       ...

H = Σ Jᵢᵀ · Wᵢ · Jᵢ     ← 信息矩阵, 各因子累加
g = Σ Jᵢᵀ · Wᵢ · bᵢ     ← 梯度向量

解 H·Δx = -g             ← 大型稀疏线性系统

每个因子只更新 H 和 g 中涉及的那几行/列。变量的信息量 = 连接到它的所有因子的权重之和。

为什么是"稀疏"？ — YawMotionFactor 只连 3 个变量 (r_pre, w_pre, r_cur)，它对 X(k) 的雅可比为 0。所以 J 矩阵大部分是零。H = JᵀJ 也是稀疏的。利用稀疏性 → O(n) 而非 O(n³)。

5. iSAM2 的核心：贝叶斯树 + 增量更新

全图重新优化每次都要解一次 → 太慢。iSAM2 把计算分解成团 (Clique) 的树状结构，只更新受影响的部分。

5.3 重线性化

每个团有线性化点 θ₀。当变量偏离 θ₀ 超过阈值：

if |θ - θ₀| > relinearizeThreshold (0.1):
    标记该团 → 下次增量更新时, 在当前位置重新计算雅可比

这是 iSAM2 和 EKF 的最本质区别：EKF 做一次线性化后不再回头；iSAM2 可以回头重算。

冷启动为什么需要？ — 前几帧只有一两个观测，约束远远不够。直接优化 → 信息矩阵奇异 → 发散。等累积了足够的约束再第一次优化。
滑动窗口为什么需要？ — 图无限增长 → 内存↑、线性化点过时、增量更新复用率下降。定期 reset 用当前估计做先验。

9. 关键公式速查

Gauss-Newton 更新

Δx = -(JᵀWJ)⁻¹ JᵀW · error(x₀)       ← 迭代直到收敛
x₁ = x₀ ⊞ Δx                            ← ⊞ 是流形上的 retract

iSAM2 增量更新

BayesTree.insert(factor)   → 标记受影响团
BayesTree.eliminate()      → 从叶向根消元 (仅受影响链)
BayesTree.backsub()        → 从根向叶回代
Δ = new_estimate ⊟ old_estimate
if |Δ| > threshold → 标记重线性化

noiseModel 类型全解

噪声模型决定因子在优化中的权重。它的输入是标准差 σ，内部自动转为信息矩阵 Λ = 1/σ² 供 Gauss-Newton 使用。

类型构造适用场景代码示例
Diagonal::Sigmas 每个分量独立 σ 多维误差, 各分量噪声不同 Diagonal::Sigmas(Vector3(0.01, 0.03, 0.005))
Diagonal::Variances 用方差 σ² 指定 已有协方差数据 Diagonal::Variances(Vector3(1e-4, 9e-4, 25e-6))
Diagonal::Precisions 用信息 1/σ² 指定 已有信息矩阵 Diagonal::Precisions(Vector3(10000, 1111, 40000))
Isotropic::Sigma 所有分量相同 σ 1D 误差 (yaw, vyaw, 半径) Isotropic::Sigma(1, 0.005)
Unit σ=1 固定 快速原型, 测试 Unit::Create(3)
Gaussian::Covariance 完整协方差矩阵 有相关性的噪声 (CWNA Q 矩阵) Gaussian::Covariance(Q_6x6)
Robust 套一层 M-estimator 抗外点 (反光, 遮挡) Robust::Create(Huber::Create(1.345), noise)

三种指定方式的关系

σ (标准差)  —最直观→  Sigmas(0.01)
σ² (方差)   —中间量→  Variances(0.0001)
1/σ² (信息) —内部用→  Precisions(10000)

关系: precision = 1/variance = 1/σ²
      σ 越小 → 信息越大 → 因子越"强"

Robust 噪声 — 抗外点

标准最小二乘 (L2) 对异常值极度敏感: 一个错误角点能拖偏整个优化。

损失函数:
  L2:     ρ(e) = ½e²                  ↗ 二次增长, 外点权重极大
  Huber:  ρ(e) = ½e²        |e|≤k     → 小误差用二次
               = k|e|-½k²   |e|>k     → 大误差退化为线性
  Cauchy: ρ(e) = ½log(1+(e/k)²)       → 权重随误差递减
  Tukey:  ρ(e) = 完全截断 |e|>k        → 大误差直接忽略

使用示例:
  auto base = Diagonal::Sigmas(Vector2(1.0, 1.0));
  auto robust = Robust::Create(Huber::Create(1.345), base);
  // 效果: 像素误差 > 1.345σ 时从 L2 退化为 L1, 不再被外点拖偏
为什么不直接用 Robust？ 当前仿真数据没有外点 (所有角点来自 PnP 真值+高斯噪声)。 如果未来接入真实检测器 (可能有反光/遮挡导致的错误角点), 给 ArmorReprojFactor 套 Huber 能显著提升鲁棒性。但注意: Robust 噪声使优化问题非凸, 需要更好的初值。

信息矩阵 vs 协方差

信息矩阵 Λ = JᵀWJ     ← 优化用的是这个
协方差   Σ = Λ⁻¹      ← 不确定性 (KF 的 P 矩阵)
边际协方差 Σ_ii = (Λ⁻¹)_ii    ← 变量 i 的边际不确定性
协方差从哪来？ — iSAM2 可以从贝叶斯树直接计算任意变量的边际协方差 marginalCovariance(key)，供马氏距离匹配使用。EKF 的 P 矩阵天然就是协方差。

11.2 步骤 1-2: 协方差时间传播 (CWNA)

cov_X_pred = X_cov + V_cov·dt² + Q_translation·dt²
cov_R_pred = R_cov + W_cov·dt² + Q_yaw·dt²

center_cov(4×4) = ┌                ┐
                   │ cov_X_pred  0 │
                   │     0    cov_R│
                   └                ┘

上一帧 ISAM2 提取的后验协方差 (X_cov, V_cov, R_cov, W_cov) 通过 CWNA 运动模型向前传播 dt 秒：

X(k) = X(k-1) + V(k-1)·dt + ½a·dt²

∂X(k)/∂X(k-1) = I    (位置对位置)
∂X(k)/∂V(k-1) = dt·I (位置对速度)
∂V(k)/∂V(k-1) = I    (速度对速度)

Var[X(k)] = Var[X(k-1)] + dt²·Var[V(k-1)] + dt²·Q_xx
Var[R(k)] = Var[R(k-1)] + dt²·Var[W(k-1)] + dt²·Q_rr

11.3 步骤 3: 第一跳 — 中心 → 装甲板位置

装甲板位置是中心位置 + yaw 的非线性函数：

armor_x = center_x - r·cos(yaw + I·π/2)
armor_y = center_y - r·sin(yaw + I·π/2)
armor_z = center_z + dz_armor

雅可比 J(3×4) 对 [center_x, center_y, center_z, center_yaw]：

     ┌                         ┐   ┌                        ┐
     │ 1  0  0  r·sin(yaw+Iπ/2) │   │ 1  0  0  -offset.y │
J =  │ 0  1  0 -r·cos(yaw+Iπ/2) │ = │ 0  1  0   offset.x │
     │ 0  0  1        0         │   │ 0  0  1      0       │
     └                         ┘   └                        ┘

为什么 ∂x/∂yaw = -offset.y？
  offset = armor - center = [-r·cos(φ), -r·sin(φ), dz]
  ∴ r·sin(φ) = -offset.y  →  直接用已知 offset 算, 无需求三角函数
通用协方差传播公式: P_target = J · P_source · Jᵀ
4×4 中心协方差通过 3×4 雅可比变换为 4×4 装甲板位置协方差。

11.4 步骤 4: 第二跳 — 笛卡尔 → YPD 球坐标

yaw_sph  = atan2(y, x)
pitch    = atan2(z, √(x²+y²))
distance = √(x²+y²+z²)

J_sph = ∂[yaw_sph, pitch, distance] / ∂[x, y, z]  (3×3 解析雅可比)

两跳联动:

P_ypd = J_sph · J_xyz · P_center · J_xyzᵀ · J_sphᵀ
      = J_total · P_center · J_totalᵀ
为什么用 YPD 空间而不是笛卡尔空间做匹配？ 观测噪声在 YPD 空间是自然解耦的：yaw 误差 ≈ 角度误差，distance 误差 ≈ 深度误差。 在笛卡尔空间，同样的角度误差在不同距离产生的位置误差不同，协方差矩阵有复杂相关性。 YPD 空间的对角噪声假设更接近真实 PnP 误差特性。

11.5 步骤 5: 自适应观测噪声 S = P + R

观测噪声 不是常数 — 取决于装甲板的视角和距离：

噪声分量公式物理含义
yaw_sph 常数 1.0 PnP 角度噪声，与视角无关
pitch 常数 1.0 同上
distance log(0.1·|倾角|+1) + 0.001 斜着看装甲板 → PnP 深度退化 → 距离噪声指数增长
armor_yaw log(距离+1)/20 + 0.01 越远装甲板朝向越难从像素判断
为什么用 log 缩放？ PnP 在极端条件下的退化不是线性的。从 0°→10° 倾角和从 60°→70° 倾角，噪声增长量级不同。 log 函数在大值区间压缩增长速率，能更好地匹配 PnP 的实际协方差膨胀曲线。

11.6 步骤 6: 马氏距离 + Unique Match

对每对 (预测装甲板, 观测) 计算马氏距离:

error(4D) = [yaw_sph_obs-pred, pitch_obs-pred, dist_obs-pred, armor_yaw_obs-pred]
d² = errorᵀ · S⁻¹ · error         ← 马氏距离平方

if d² < max_match_mahalanobis_distance (20.0):
    accept

4 个容器 (armor 0/1/2/3)，每容器保留最小马氏距离的观测：

观测 A: armor_0 d²=0.5, armor_1 d²=3.2, armor_2 d²=8.1, armor_3 d²=12.0
                                                            ↓ 各取 d²<20 的最小
观测 B: armor_0 d²=9.0, armor_1 d²=0.3, armor_2 d²=7.5, armor_3 d²=11.0

结果:
  bucket[0] = {d²=0.5, obs_A}   ← 唯一竞争者, 匹配
  bucket[1] = {d²=0.3, obs_B}   ← 唯一竞争者, 匹配
  bucket[2] = {d²=7.5, obs_B}   ← < 20, 匹配
  bucket[3] = 空                 ← 无匹配
注意：不是匈牙利算法！ 这是简化的贪心策略。当两个观测竞争同一装甲板时，只保留更近的那个。 一个观测可以同时匹配多个装甲板（只要马氏距离都足够小）。 这对 4 装甲板场景足够——伪匹配会被几何约束因子在优化阶段自动排除。

11.7 马氏距离 vs 欧几里得距离

欧几里得距离马氏距离
定义 d = √(Δx² + Δy² + Δz²) d² = errorᵀ·S⁻¹·error
权重 所有方向等权重 S⁻¹ 加权：不确定方向宽容，确定方向严格
门限含义 "距离差 < N 米" "在正态分布里离均值 < N 个标准差"
例子 2m 远的匹配和 0.2m 的错配门槛相同 远距离有更大不确定 → 自动放宽门槛
马氏距离的本质： 把"这个观测偏离预测多远"换算成"在这个正态分布里发生概率多大"。 d²=1 → 约 1σ → 高概率匹配 (68%)。 d²=4 → 约 2σ → 边界匹配 (95%)。 d²=20 → 远超 3σ → 几乎不可能是正确匹配。
